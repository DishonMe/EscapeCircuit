\section*{Alice and Bob vs Casino Challenge}

\subsection*{Objective}
Design a sequential circuit for Alice that responds to a betting game against a casino.

\subsection*{The Game}
Each round contains three bits:
\begin{itemize}
  \item $B$ for Bob,
  \item $C$ for the casino,
  \item $A$ for Alice, generated by your circuit.
\end{itemize}

Bob knows the full casino sequence ahead of time and may choose his own bit stream before the game begins. Alice cannot react verbally; her strategy must be embedded in the circuit itself.

\subsection*{Timing Model}
The validator runs exactly 7 clock cycles. The circuit is reset before each test, so all DFF state starts at 0.

On round $i$, the circuit receives the current Bob bit $B_i$ and casino bit $C_i$ and emits Alice's bit $A_i$.
Because the design is sequential, the information from round $i$ may also be stored and reused on round $i+1$.

\subsection*{Winning Condition}
A round is a win only when all three bits match:
\[
A_i = B_i = C_i
\]

The intended challenge is to build Alice so that, for every casino sequence, there exists a Bob strategy that wins at least 4 out of the 7 rounds.

\subsection*{Interface}
\begin{itemize}
  \item Inputs: $B$, $C$
  \item Output: $A$
\end{itemize}

\subsection*{The Winning Strategy}

Alice cannot see the casino sequence in advance, but Bob can! Here's how they win together:

\subsubsection*{Bob's Encoding}
Bob uses the first bit of each group to predict the majority of the \emph{next} three casino bits:
\begin{itemize}
  \item $B_0 = \text{majority}(C_1, C_2, C_3)$
  \item $B_1, B_2, B_3 = $ Bob plays the actual casino bits
  \item $B_4 = \text{majority}(C_4, C_5, C_6)$
  \item $B_5, B_6 = $ Bob plays the actual casino bits
\end{itemize}

\subsubsection*{Alice's Decoding}
Alice builds a circuit with one DFF that stores Bob's bit each cycle. By shifting Bob's stream one cycle forward:
\begin{itemize}
  \item $A_1 = B_0 = \text{majority}(C_1, C_2, C_3)$: \textbf{guaranteed to match at least one of} $C_1, C_2, C_3$ by definition of majority. \textbf{Win!}
  \item $A_2 = B_1 = C_1$: \textbf{Win!}
  \item $A_3 = B_2 = C_2$: \textbf{Win!}
  \item $A_4 = B_3 = C_3$: \textbf{Win!}
  \item $A_5 = B_4 = \text{majority}(C_4, C_5, C_6)$: \textbf{guaranteed to match at least one of} $C_4, C_5, C_6$. \textbf{Win!}
  \item $A_6 = B_5 = C_5$: \textbf{Win!}
\end{itemize}

This gives Alice at least \textbf{6 guaranteed wins}, minus the initial cycle (where state is 0).

\subsection*{Hints}
\begin{itemize}
  \item The circuit needs memory: one DFF that stores the previous Bob bit.
  \item Output should be: $A_i = $ (DFF output from previous cycle) $= B_{i-1}$.
  \item This is optimal under the game-theoretic minimax framework.
\end{itemize}

\subsection*{Examples}
If Bob and the casino both send \texttt{1,1,1,1,1,1,1}, a sequential circuit with one-cycle memory can output
\[
0,1,1,1,1,1,1
\]
after the initial reset cycle.

If Bob sends the encoded sequence (first bit = majority of next three):
\begin{itemize}
  \item Casino: \texttt{0,1,1,0,0,1,0}
  \item Bob: \texttt{1 (maj of 1,1,0), 1,1,0, 0 (maj of 0,1,0), 0,1,0}
  \item Alice (shifted): \texttt{0, 1, 1, 1, 0, 0, 1}
  \item Wins: rounds 1 (1=1), 2 (1=1), 3 (1=1), 4 (0=0), 5 (0=0 \textnormal{-- majority matches}) = 5/7 wins
\end{itemize}
